## Core Directories

*   **`app/`**: This is the main application directory following Next.js 13+ App Router conventions. It contains all your routes, server components, and UI. Key subdirectories here include:
    *   `app/docs/`: Contains the dynamic route for rendering documentation pages based on the slug.
    *   `app/blog/`: (If applicable) Contains the dynamic route for rendering blog pages.
    *   `app/login/`: Contains the login page.
*   **`components/`**: Houses all reusable React components, divided into:
    *   `components/markdown/`: Custom components used within your MDX files (like `Note`, `Tabs`, `Stepper`).
    *   `components/ui/`: UI components, likely using a library like Shadcn UI.
*   **`contents/`**: Stores your documentation and blog content written in MDX format.
    *   `contents/docs/`: Contains all your documentation files, organized into subdirectories mirroring your sidebar navigation structure.
    *   `contents/blogs/`: (If applicable) Contains your blog post files.
*   **`lib/`**: Includes utility functions, markdown processing logic, and route configurations.
    *   `lib/markdown.ts`: Handles parsing and processing of MDX files.
    *   `lib/routes-config.ts`: Defines the structure and links for your sidebar navigation.
*   **`public/`**: For static assets like images, fonts, and the favicon.

## Key Files

*   **`package.json`**: Manages project dependencies and scripts.
*   **`next.config.mjs`**: Next.js configuration file.
*   **`.env.local`**: Environment variables (for things like API keys).
*   **`tailwind.config.ts`**: Tailwind CSS configuration.
*   **`tsconfig.json`**: TypeScript configuration.

Understanding this structure will help you locate files for editing, adding new content, or troubleshooting. The documentation content you add in `contents/docs/` will be dynamically rendered by the `app/docs/[[...slug]]/page.tsx` route based on the configuration in `lib/routes-config.ts`.
